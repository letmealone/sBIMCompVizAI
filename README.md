# SBIM CompViz AI — IFC 평면도 비교 도구

> 전문가가 만든 IFC와 AI가 생성한 IFC를 층/공간 단위로 자동 매칭하고, 평면도로 시각 비교하며,
> 부재(벽·기둥·문·창·설비 등) 구성 차이를 엑셀로 정량 비교하는 Streamlit 앱입니다.

이 문서는 비전문가(비-BIM/비-개발) 동료 연구자도 전체 흐름을 이해할 수 있도록,
"무엇을 입력하면 → 어떤 과정을 거쳐 → 무엇이 출력되는지"를 중심으로 정리했습니다.

---

## 1. 이 도구가 하는 일 (한 줄 요약)

**"같은 건물"을 그린 두 개의 IFC 파일(전문가용/AI 생성용)을 넣으면, 층과 공간을 자동으로 짝지어
나란히 비교해주고, 그 공간에 어떤 부재가 얼마나(개수·면적) 있는지까지 비교해줍니다.**

```
[전문가 IFC]        [AI IFC]
      │                 │
      └────────┬────────┘
               ▼
   ① 층 자동 매칭 (표고 또는 AI/LLM)
               ▼
   ② 공간 자동 매칭 (면적 + 위치 유사도)
               ▼
   ③ 화면에서 평면도 나란히 보기 (클릭 / hover / 범례 필터)
               ▼
   ④ 엑셀로 정량 비교 결과 내려받기 (zip)
```

---

## 2. 입력

| 입력 | 설명 |
|---|---|
| **IFC 파일 2개** | "전문가(A)"와 "AI(B)" — 같은 건물을 각각 다른 모델러/도구로 작성한 `.ifc` 파일 |
| **공간 매칭 임계값** (옵션) | 면적 오차 허용치(㎡, 기본 2.0), 공간 중심점(centroid) 거리 허용치(m, 기본 1.0) |
| **AI 층 매핑 사용 여부** (옵션) | 켜면 Google Gemini(`gemini-3.1-flash-lite`)로 층 이름·표고·공간구성 유사도를 종합 판단해 매칭. 끄면 표고(고도)만으로 자동 매칭 |
| **Google AI API Key** (옵션) | AI 층 매핑 사용시에만 필요. `.streamlit/secrets.toml`에 등록하거나 화면에서 직접 입력 |

---

## 3. 전체 처리 파이프라인

### 3.1 IFC 파싱 및 사전계산
파일을 올리면 즉시 층·공간·벽 내/외부 판정을 계산하고, **층별 부재 지오메트리를
사전계산**해 캐싱합니다(업로드 직후 한 번만 진행률 표시와 함께 시간이 걸리고, 이후
공간 클릭은 즉시 반응합니다).

### 3.2 층 매칭
두 방식 중 선택:
- **표고 기반(기본)**: 각 층의 표고(고도) 차이를 통계적으로 비교해 자동 매칭
- **AI(Gemini) 기반(옵션)**: 층 이름의 문맥(예: "1층"="Level 1"="1F"), 표고, 그 층의
  공간 구성(방 종류·개수)까지 종합 판단. **IFC 파일쌍 하나당 API를 정확히 1회만 호출**
  하며, 분당 요청/토큰 한도(RPM/TPM)를 자동으로 지키도록 대기·재시도가 내장되어 있습니다.
  일일 호출 한도(RPD)는 API 키/프로젝트 단위라 앱이 직접 통제할 수는 없습니다.

### 3.3 공간 매칭
매칭된 층 쌍마다, 공간의 **면적**과 **중심점 위치**(좌표계 평행이동 보정 포함)가
임계값 이내로 가까우면 같은 공간으로 간주해 번호 배지(①②③...)를 붙여 양쪽 평면도에
표시합니다.

### 3.4 공간에 접한 부재 찾기 (관계 + 지오메트리 보강, 2단계)
IFC 표준상 "이 벽이 이 공간과 접해있다"는 정보는 `IfcRelSpaceBoundary`라는 관계로
기록되는데, 실제 파일에서 이 관계가 **누락**되어 있거나(예: 두 공간이 공유하는 벽인데
한쪽 공간과의 관계만 기록됨), 부재가 층에 직접 연결되지 않고 **조립체 하위부품**으로만
연결된 경우(예: 계단 조립체 속의 계단참)가 확인되어, 관계만으로는 부재를 놓치는 경우가
있습니다. 이를 보완하기 위해:
1. 관계(`RelSpaceBoundary`)로 우선 탐색
2. 관계로 못 찾은 부재도, 같은 층 안에서 **실제로 맞닿아 있는지(지오메트리 거리)**를
   추가 검사해 보완
3. 층 소속 조회 자체도 **하위 조립체까지 재귀적으로** 따라가며 부재를 수집

### 3.5 벽 내/외부 판정 (2단계 자동 폴백)
> ⚠️ 참고: `Pset_*Common.IsExternal` 속성을 신뢰할 수 없는 IFC가 다수 확인되어(속성은
> 존재하지만 실제 값이 사실과 다른 경우), 이 값에 의존하는 판정 단계는 현재
> **비활성화**되어 있습니다(코드는 남겨두고 실행만 꺼둠).

1. **1차(현재 사용)**: 벽에 접한 공간들의 경계면 좌표를 분석해 "양쪽 면 모두에 공간이
   있는지" 확인 → 있으면 내벽, 한쪽만 있으면 외벽으로 추정
2. **2차(추정, 근거 가장 약함)**: 좌표 정보 자체가 없으면, 서로 다른 공간 2개 이상과
   관계가 있는지만으로 내벽 추정 — 결과에 "(추정)" 표시

### 3.6 면적 산정 (다단계 자동 폴백 — 이 도구의 핵심)
공간 하나에 접한 벽의 면적을 "그 공간 몫"만큼 나누는 문제를, 데이터 품질에 따라
아래 순서로 더 정밀한 방법을 자동으로 시도합니다.

| 단계 | 방법 | 필요 데이터 |
|---|---|---|
| 1차 | `RelSpaceBoundary`의 경계면 좌표를 직접 적분해 정밀 계산 + 정합성 자동검증(벽이 꺾인 경우까지 고려) | ConnectionGeometry(2단계 경계 좌표) |
| 2차 | 화면에 보여주는 것과 **동일한** 벽 구간 클리핑 결과의 "길이 비율"을 실제 벽면적(`Gross_Side_Area`)에 곱함 | ConnectionGeometry 또는, 없으면 공간·벽 폴리곤의 변(edge) 겹침 |
| 3차 | 공간 footprint를 살짝 부풀려 벽과 겹치는 비율로 근사 | 공간·벽의 자체 형상(항상 존재) |
| 보조 | 벽 자체 면적값(`Gross_Side_Area`)이 없거나 빈 문자열 등으로 비어있으면, 문/창/커튼월에 쓰던 방식과 동일하게 전체 bounding 치수(폭×높이)로 자동 대체 | 벽 지오메트리 |

**중요**: 평면도에 표시되는 "실제 접한 구간" 하이라이트와 엑셀에 찍히는 면적 숫자는
**같은 계산 결과를 공유**하도록 만들어 두 값이 항상 일치합니다.

### 3.7 평면도 시각화
- 공간 내부 클릭(또는 드롭다운 선택) → 상세 정보 표시
- 자동 매칭된 공간은 옅은 색, **선택된 공간은 접한 벽/부재/설비가 카테고리별로 하이라이트**
  — 벽 이외 부재(기둥·문·창·바닥·지붕·계단 등)와 설비(조명·센서·소방장치·경보기 등)를
  하나로 묶지 않고 **개별 클래스마다 각자의 색**으로 구분해 보여줍니다.
- **범례가 곧 필터입니다**: 범례에서 항목을 고르면(멀티셀렉트) 선택한 카테고리만
  양쪽 평면도 모두에서 강조 표시되고, 나머지는 배경처럼 옅게 처리됩니다.
- 기본(선택 없음) 상태의 배경 채색은 "존재감 정도"만 인식되도록 투명도를 낮게 유지합니다.
- 벽은 전체가 아니라 **실제로 접한 부분만** 진하게 표시(나머지는 옅게, 맥락용)
- 부재에 마우스를 올리면 치수·면적·재질·내외벽 판정 정보 표시, 클릭 가능한 영역은
  커서가 손가락 모양으로 바뀝니다.

## 4. 출력

### 4.1 화면 (Streamlit)
두 IFC의 선택된 층 평면도를 나란히 표시하고, 클릭한 공간의 면적·접한 부재 요약·
내외벽 구분·산출방식(정밀/추정 여부)을 함께 보여줍니다.

### 4.2 엑셀 (zip 파일 하나로 다운로드)

파일명에는 실제 업로드한 IFC 파일명이 그대로 반영됩니다(예: 전문가 파일이
`SB_실증.ifc`, AI 파일이 `AI_모델.ifc`라면 zip 이름은 `SB_실증_AI_모델_IFC비교.zip`).

| 파일 | 내용 |
|---|---|
| `01_{전문가파일명}_추출.xlsx` | 전문가 IFC의 전체 부재 목록/속성 — 계층구조, 클래스별 상세(치수 포함), 통합조회, 공간-부재 매칭, 벽 내외벽 판정 등 |
| `02_{AI파일명}_추출.xlsx` | AI IFC의 전체 부재 목록/속성 (동일 형식) |
| `03_{전문가파일명}_{AI파일명}_비교결과.xlsx` | 아래 7개 시트로 구성된 비교 결과 |

`03_..._비교결과.xlsx`의 시트 구성:

| 시트명 | 내용 |
|---|---|
| `01_층_매칭` | 층 매핑 테이블 (AI 매핑시 확신도·판단근거 포함) |
| `02_공간_매칭` | 전체 층에 걸친 공간 매칭 테이블 (면적차/centroid거리 포함) |
| `03_구조부재_내외구분비교` | 매칭된 공간마다 벽·기둥·문·창 등을 **내부/외부/판정불가**로 나눠 개수·면적 비교 |
| `03b_구조부재_통합비교` | 위와 같은 항목을 내/외부 구분 없이 클래스별로만 비교 |
| `03c_층단위_부재비교` | 공간 매칭과 무관하게 **층 전체** 단위로 부재 개수·면적 비교(계산량 적음) |
| `04_설비_비교` | 조명/센서/소방장치/경보기 등 설비 개수 비교 (구조부재와 데이터 출처가 달라 별도 시트) |
| `05_미매칭공간_객체요약` | 대응 공간을 못 찾은 공간들의 부재 구성 요약 |

## 5. 알려진 한계

- **IFC 데이터 품질에 크게 좌우됩니다.** 특히 `RelSpaceBoundary`의 2단계 경계 좌표
  (`ConnectionGeometry`)가 없는 파일에서는 근사 방법으로 자동 전환되며, 이 경우 결과의
  정밀도가 다소 낮아질 수 있습니다(화면·엑셀 모두 어떤 방식으로 계산됐는지 표시됨).
- **`03_구조부재_내외구분비교`(공간별)와 `03c_층단위_부재비교`(층 전체)의 면적 합계는
  일치하지 않을 수 있습니다.** 내벽은 양쪽 공간에서 각각 카운트되는 반면, 층단위는
  부재당 한 번만 세기 때문으로, 서로 "다른 질문에 답하는" 지표입니다(오차 아님).
- **`Pset_*Common.IsExternal` 속성은 신뢰도 문제로 현재 벽 내/외부 판정에 사용하지
  않습니다.** 관계·지오메트리 기반 판정만 사용하므로, 일부 벽은 "(추정)"으로 표시됩니다.
- **AI 층 매핑은 무료 API 할당량(분당 요청 수 등) 안에서만 재시도합니다.** 일일 한도는
  API 키/프로젝트 단위라 이 앱이 직접 통제할 수 없습니다.

---

## 6. 빠른 시작

1. Streamlit 앱 접속 → 전문가 IFC, AI IFC 각각 업로드
2. (자동) 층별 부재 지오메트리 사전계산 진행률 표시 → 완료 대기
3. 사이드바에서 층 선택(자동 매칭됨) → 평면도에서 공간 클릭
4. 필요시 사이드바 "공간 자동 매핑", "AI 층 매핑" 옵션 켜기
5. 범례에서 보고 싶은 부재/설비 종류만 골라 강조 표시 가능
6. "📥 비교 엑셀(zip) 생성" 버튼 클릭 → 다운로드
